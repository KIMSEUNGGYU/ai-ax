# session-manager 사용 가이드

> v2.0.0 | 세션 간 맥락 유지 + 지식 영속 저장

## 기능 요약

| 기능 | 설명 |
|------|------|
| `/save` | 작업 진행 상태를 `.ai/current.md`에 저장 |
| `/note` | 세션 지식을 영속 저장소에 보존 (AI가 저장 위치 자동 판단) |
| `/resume` | Hook 미작동 시 current.md 수동 로드 |
| `note` 스킬 | "정리해줘", "남겨둬" 등 자연어로 `/note` 트리거 |
| SessionStart Hook | 세션 시작 시 current.md 자동 로드 |
| SessionEnd Hook | 세션 종료 시 current.md 타임스탬프 자동 갱신 |

## 빠른 시작

```
세션 시작 → (자동) 이전 작업 로드
작업 중   → /save로 진행 상태 저장
          → /note로 지식 영구 보존
세션 종료 → 그냥 나가면 됨
```

## 커맨드

### `/save` — 작업 상태 저장

현재 작업의 진행 상태를 `.ai/current.md`에 저장한다.

**언제 쓰나:**
- 세션 중간에 체크포인트를 남기고 싶을 때
- 세션 종료 전에 진행 상태를 기록할 때
- 다음 세션에서 이어서 할 작업이 있을 때

**저장 내용:**
- 목표, 진행 체크리스트, 작업 내역, 결정사항, 메모

**사용법:**
```
/save              ← current.md 생성 또는 업데이트
"현재 상태 저장해"  ← 같은 동작
```

**참고:**
- 단일 세션으로 끝나는 작업이면 안 써도 됨
- current.md 삭제는 사용자가 직접 수행 (작업 완료 시)

---

### `/note` — 지식 영속 저장

세션 대화에서 나온 지식을 영구 저장소에 보존한다.

**언제 쓰나:**
- 분석 결과를 남기고 싶을 때
- 배운 개념을 정리하고 싶을 때
- 재사용할 코드 패턴을 발견했을 때
- 트러블슈팅 경험을 기록하고 싶을 때

**사용법:**
```
/note                    ← "어떤 내용을 정리할까요?" 질문
/note API 구조 정리해줘   ← 해당 내용 바로 정리
"이거 정리해줘"           ← note 스킬 트리거 (같은 동작)
"남겨둬"                  ← note 스킬 트리거
"패턴으로 저장해줘"        ← note 스킬 트리거
```

**저장 위치 (AI 자동 판단 → 사용자 확인):**

| 내용 성격 | 저장 위치 | 예시 |
|----------|----------|------|
| 프로젝트 설계/분석/결정 | `.ai/notes/` | API 구조 분석, 아키텍처 결정 |
| 재사용 코드 패턴 | `.ai/patterns/` | 커스텀 훅 패턴, 에러 처리 패턴 |
| 개인 학습 | `~/obsidian-note/00_Inbox/` | TIL, 개념 정리, 트러블슈팅 |

**동작 흐름:**
1. 정리할 내용 파악
2. AI가 저장 위치 자동 판단
3. "`.ai/notes/api-design.md`에 저장할게요" → 사용자 확인
4. 기존 파일 있으면 업데이트, 없으면 새로 생성

---

### `/resume` — 수동 복원

SessionStart Hook이 작동하지 않았을 때 수동으로 이전 작업을 로드한다.

**사용법:**
```
/resume    ← current.md 수동 로드
```

보통은 쓸 일 없음. Hook이 자동으로 처리함.

---

## Hook (자동 동작)

### SessionStart — 세션 시작 시

- `.ai/current.md`가 있으면 자동으로 내용을 로드
- Claude에게 이전 작업 컨텍스트 주입
- 사용자에게 "이어서 할까요?" 확인

### SessionEnd — 세션 종료 시

- `.ai/current.md`가 있으면 last-active 타임스탬프 자동 갱신
- 내용 자체를 업데이트하지는 않음 (그건 `/save`의 역할)

## /save vs /note 비교

| | `/save` | `/note` |
|------|---------|---------|
| **목적** | 작업 진행 상태 기록 | 지식 영구 보존 |
| **저장 위치** | `.ai/current.md` | notes / patterns / obsidian |
| **수명** | 작업 완료 시 삭제 | 영구 |
| **빈도** | 세션당 1-2회 | 필요할 때 |
| **예시** | "3단계까지 완료, 4단계 남음" | "이 API 구조 분석 결과" |

## 저장 체계

```
세션 대화 (휘발)
    ↓ /save (수동)
.ai/current.md (작업 상태, 작업 완료 시 삭제)
    ↓ /note (수동)
영속 저장소:
  ├── .ai/notes/       ← 프로젝트 지식
  ├── .ai/patterns/    ← 재사용 패턴
  └── ~/obsidian-note/ ← 개인 학습
```

## 실전 시나리오

### 멀티 세션 기능 구현

```
세션 1:
  → (자동) "새 세션입니다"
  → 인증 시스템 설계 작업
  → "JWT vs 세션 비교 정리해줘" → /note → .ai/notes/auth-comparison.md
  → /save → current.md 생성 (목표, 진행 상태)
  → 종료

세션 2:
  → (자동) current.md 로드 → "인증 시스템 설계 이어갈까요?"
  → 구현 작업
  → "이 토큰 갱신 패턴 남겨둬" → /note → .ai/patterns/token-refresh.md
  → /save → 진행 상태 업데이트
  → 작업 완료 → current.md 삭제
```

### 단일 세션 학습

```
  → React 19 새 기능 학습
  → "이거 정리해줘" → /note → ~/obsidian-note/00_Inbox/react-19.md
  → current.md 불필요 (단일 세션)
```

### /save 안 하고 종료

```
  → 작업 중 그냥 종료
  → SessionEnd Hook: 타임스탬프만 기록
  → 다음 세션: current.md는 마지막 /save 시점 상태로 로드
  → 팁: 주요 진전이 있었으면 종료 전 /save 습관
```

## FAQ

**Q: current.md는 언제 삭제하나요?**
작업이 완전히 끝났을 때 사용자가 직접 삭제합니다. Claude가 임의로 삭제하지 않습니다.

**Q: /note 없이 그냥 "정리해줘"라고 해도 되나요?**
네. `note` 스킬이 자연어를 인식하여 `/note`와 동일하게 동작합니다.

**Q: /note가 저장 위치를 잘못 판단하면?**
저장 전에 항상 사용자에게 확인을 받습니다. 다른 위치를 지정할 수 있습니다.

**Q: 패턴 저장은 어떻게 하나요?**
`/note`를 사용하면 됩니다. AI가 내용이 재사용 패턴이라고 판단하면 `.ai/patterns/`를 제안합니다.
