## 역할
작업 내용을 정리하고, AI 협업 패턴을 분석해서 개선점을 제시.

## 입력
- git staged 또는 현재 브랜치 작업 내용 (기본)
- 대화에서 논의한 설계/문제 해결 컨텍스트
- $ARGUMENTS (추가 지시사항)

## 수행

### Part 1: 작업 정리 (기존)
1. git diff/staged 확인
2. 대화 컨텍스트에서 설계 결정/문제 해결 내용 파악
3. md 파일로 정리

### Part 2: AI 협업 패턴 분석 (신규)
1. `.ai/logs/sessions/` 에서 최근 세션 로그 확인 (있으면)
2. 현재 대화 컨텍스트와 함께 AI 협업 패턴 분석
3. 잘한 점 / 개선할 점 식별

## 출력 (md 파일)

### 작업 정리
- 왜 이렇게 설계했는지
- 어떤 문제를 어떻게 해결했는지
- 주요 결정 사항과 이유
- 작업 내용에 따라 유동적

### AI 협업 분석 (로그 있을 때)
- 이번 세션 워크플로우 요약
- 잘 활용한 패턴 (예: 설계 → 구현 → 리뷰 순서 준수)
- 개선 가능한 패턴 (예: 리뷰 없이 PR 생성)
- 이전 세션 대비 변화 (로그 2개 이상일 때)

## 대화 출력
md 파일 저장 후, **요약만** 대화에 출력.
상세 내용은 md 파일 참조.

## 원칙
- 코드 복붙 최소화 (경로와 설명 위주)
- 결정 이유 필수
- 원인/문제, 액션, 결과 필수
- 다른 개발자도 이해 가능하게
- AI 협업 분석은 로그가 있을 때만 (없으면 Part 1만 실행)
