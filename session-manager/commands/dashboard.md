## 역할
습관 점수와 세션 로그를 분석해서 코딩 습관 대시보드를 보여준다.

## 수행

### 1. 데이터 수집
- `.ai/logs/habit-score.md` 읽기 (전체 점수 테이블)
- `.ai/logs/sessions/` 에서 최근 5개 세션 로그 읽기 (날짜 역순)

### 2. 분석 및 출력

데이터가 없으면:
```
아직 기록된 세션이 없습니다. /wrap 으로 세션을 마무리하면 습관 점수가 기록됩니다.
```

데이터가 있으면 아래 형식으로 대화에 출력:

```markdown
## 습관 대시보드

### 점수 추이 (최근 5세션)

| 날짜 | FE 품질 | AI 협업 | 메모 |
|------|---------|---------|------|
| ... | ... | ... | ... |

### 강점
- {2점 이상 꾸준히 획득하는 항목}

### 약점
- {0점이 반복되는 항목}

### 반복 지적 패턴
- {세션 로그에서 반복 등장하는 개선점}

### 다음 세션 집중 포인트
- {가장 개선 효과가 클 항목 1~2개, 구체적 행동 제안}
```

## 원칙
- 파일 읽기만 (수정 없음)
- 데이터 기반 분석만 (추측 금지)
- 집중 포인트는 구체적 행동으로 (예: "리뷰 실행하기" → "/review를 PR 전에 반드시 실행")
- 대화에 직접 출력 (별도 파일 생성 안함)
