작업 컨텍스트를 `.ai/current.md`에 저장한다. 세션 중 아무 때나 호출 가능.

$ARGUMENTS

## 동작

### 인자 처리
- 스펙 파일 경로가 주어지면 → 해당 파일을 읽어서 스펙 참조 섹션에 반영
- 인자 없으면 → 대화 컨텍스트만으로 작성

### `.ai/current.md`가 없을 때
→ 대화 컨텍스트에서 아래 정보를 수집하여 새로 생성:

```markdown
# {작업 제목}

## 목표
{한 줄 요약}

## 스펙 참조
{.ai/specs/ 파일 경로. 없으면 "없음" 으로 기재}

## 진행
- [x] {완료된 것}
- [ ] {남은 것}

## 작업 내역
- {이번 세션에서 한 구체적 작업 — 파일 경로, 변경 내용 등}

## 결정사항
- {결정 + 이유}

## 메모
{블로커, 주의사항 등. 없으면 이 섹션 생략}
```

### `.ai/current.md`가 있을 때
→ 기존 내용을 읽고 아래 항목을 업데이트:
- 진행 체크리스트 (완료 항목 체크, 새 항목 추가)
- 작업 내역 (이번 세션에서 한 구체적 작업 추가)
- 결정사항 (새 결정 추가)
- 메모 (블로커/주의사항 갱신)

## 원칙
- 간결하게 (빠른 파악용)
- 코드 복붙 금지 (경로만 기록)
- 결정 이유 필수
- 항상 전체 덮어쓰기 (최신 상태 유지)
- 복잡한 작업은 작업 고유 섹션 자유롭게 추가 (예: "후보 기능", "분석 결과", "문제 인식" 등)
- current.md 삭제 금지 — 삭제는 사용자가 직접 수행

## 대화 출력
저장 완료 후 한 줄 요약만 출력:
```
.ai/current.md 저장 완료 — {작업 제목} ({완료/진행중})
```
