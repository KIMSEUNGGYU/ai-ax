작업 컨텍스트를 `.ai/active/{task}.md`에 저장한다. 세션 중 아무 때나 호출 가능.

$ARGUMENTS

## 동작

### 1. 작업 파일 결정

- 인자로 파일명이 주어지면 → `.ai/active/{파일명}.md` 사용
- 인자 없으면:
  1. `.ai/active/` 폴더 확인
  2. 파일이 1개면 → 그 파일 업데이트
  3. 파일이 여러 개면 → AskUserQuestion으로 어떤 작업인지 질문
  4. 파일이 없으면 → 대화 컨텍스트에서 작업 제목 추출 → 새 파일 생성

### 2. `.ai/active/` 폴더가 없을 때
→ 폴더 생성 후 진행

### 3. 해당 작업 파일이 없을 때
→ 대화 컨텍스트에서 아래 정보를 수집하여 새로 생성:

```markdown
# {작업 제목}

## 스펙
- {기획서 요약 or 링크}
- {주요 기능}

## 작업
- [x] {완료된 것}
- [ ] {남은 것}

## 현재 컨텍스트
{지금 하고 있는 작업의 구체적 상태}

## 결정사항
- {결정 + 이유}

## 메모
{블로커, 주의사항 등. 없으면 이 섹션 생략}
```

### 4. 해당 작업 파일이 있을 때
→ 기존 내용을 읽고 아래 항목을 업데이트:
- 작업 체크리스트 (완료 항목 체크, 새 항목 추가)
- 현재 컨텍스트 (최신 상태로 교체)
- 결정사항 (새 결정 추가)
- 메모 (블로커/주의사항 갱신)

### 5. 세션 이력 기록
- 대화 컨텍스트에서 `[session-manager] session_id: {id}` 를 찾는다
- session_id가 있으면 → 작업 파일 끝의 `## 세션 이력` 섹션에 `- {session_id} ({timestamp})` 추가
- `## 세션 이력` 섹션이 없으면 → 파일 끝에 섹션 생성
- 이미 같은 session_id가 기록되어 있으면 → 건너뛴다
- timestamp 형식: `YYYY-MM-DD HH:MM` (현재 시각)

### 6. INDEX.md 동기화
- `.ai/INDEX.md`가 있으면 → "현재 진행 중" 섹션에 해당 작업 링크 추가/업데이트
- 없으면 → 생략 (INDEX.md는 별도 생성)

## 원칙
- 간결하게 (빠른 파악용)
- 코드 복붙 금지 (경로만 기록)
- 결정 이유 필수
- 항상 전체 덮어쓰기 (최신 상태 유지)
- 복잡한 작업은 작업 고유 섹션 자유롭게 추가
- active 파일 삭제 금지 — 삭제는 `/done`으로만 수행

## 대화 출력
저장 완료 후 한 줄 요약만 출력:
```
.ai/active/{파일명}.md 저장 완료 — {작업 제목} ({완료/진행중})
```
