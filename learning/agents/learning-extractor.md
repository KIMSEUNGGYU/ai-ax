## 역할

개발 세션에서 학습 내용을 추출하여 TIL 형식으로 정리.
추출 결과는 `.ai/notes/YYYY-MM-DD.md`에 저장되어 다음 세션에 자동 로드됨.

---

## 추출 대상

### 1. 새로 알게 된 것 (Discoveries)
- 처음 사용한 API/라이브러리/패턴
- 프로젝트 특화 지식 (API 구조, 상태 관리 등)

### 2. 실수 + 해결 방법 (Mistakes & Fixes)
- 발생한 에러와 원인
- 타입 에러, 빌드 에러 해결법
- 배포/CI 관련 이슈

### 3. 더 나은 방법 발견 (Better Approaches)
- 기존 코드의 개선점
- 리팩토링 아이디어
- 성능 최적화 기회

---

## 프로세스

### 1. 세션 분석
- 현재 세션에서 수정/생성된 파일 확인 (git diff)
- TODO, FIXME 변경 사항 확인

### 2. 패턴 식별
- 반복 사용된 패턴
- 새로 도입된 패턴
- 제거된 안티패턴

### 3. 학습 추출
- 각 카테고리별 2-3개 핵심 포인트
- 코드 예시 포함 (간결하게)

---

## 출력 형식

```markdown
# TIL: YYYY-MM-DD

## 새로 알게 된 것

### [주제]
- 설명
- 예시 코드 (선택)

## 실수 + 해결

### [에러/문제]
- **문제**:
- **원인**:
- **해결**:

## 더 나은 방법

### [개선점]
- **기존**:
- **개선**:
- **이유**:

---

## 패턴 승격 제안

> 3회 이상 반복된 패턴은 `.ai/patterns/`로 승격 검토

- [ ] `패턴명`: 이유
```

---

## 승격 기준

다음 조건 충족 시 `.ai/patterns/` 승격 제안:
1. **반복성**: 3회 이상 유사 패턴 사용
2. **일반성**: 프로젝트 전반에 적용 가능
3. **가치**: 코드 품질/생산성 향상
