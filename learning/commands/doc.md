## 문서 정리 프로세스

유저가 기억/백업하고 싶은 내용을 정리하여 저장.

### 1. 주제 파악

argument로 전달된 주제 또는 대화 맥락에서 주제 파악.

### 2. 자동 분류

대화 맥락에서 문서 유형을 판단. 아래 기준으로 자동 분류:

| 신호 | 유형 |
|------|------|
| 에러/버그/디버깅/삽질 언급 | troubleshooting |
| 규칙/원칙/컨벤션/패턴 논의 | guideline |
| 개념/기술/라이브러리 학습 | tech-memo |
| 설계/아키텍처/의사결정 | decision |
| 비교/선택/트레이드오프 | comparison |
| 개념 질문/몰랐던 것/이해 요청 | til |

분류가 애매하면 유저에게 한 번만 확인. 복수 유형 가능.

### 3. 작성 원칙

템플릿 채울 때 아래 원칙 준수:

- **한 문서 한 주제** — 여러 주제가 섞이면 파일을 나눈다
- **가치 먼저** — `> 한 줄 요약`에 이 문서를 왜 읽어야 하는지 담는다
- **주체를 분명하게** — "설정해야 한다" (X) → "개발자가 .env를 설정한다" (O)
- **구체적으로** — "성능이 좋다" (X) → "응답 시간 200ms → 50ms로 개선" (O)
- **필요한 정보만** — 맥락 없이도 이해 가능한 최소 정보. 장황한 배경 설명 금지

### 4. 템플릿 적용

분류된 유형에 맞는 템플릿으로 작성:

#### troubleshooting (트러블슈팅)

```markdown
# [주제]

> 한 줄 요약

## 문제
- 증상:
- 발생 조건:

## 원인
-

## 해결
- 액션:
- 코드 변경: (있으면)

## 결과
- 해결 확인:
- 재발 방지:
```

#### guideline (가이드라인)

```markdown
# [주제]

> 한 줄 요약

## 원칙
-

## 규칙

| 규칙 | 이유 | 예시 |
|------|------|------|
|  |  |  |

## 예외
- 이 규칙을 안 따르는 경우:
```

#### tech-memo (기술 메모)

```markdown
# [주제]

> 한 줄 요약

## 핵심 개념
-

## 사용법
```코드 예시```

## 주의점
-
```

#### decision (설계 결정)

```markdown
# [주제]

> 한 줄 요약

## 배경
- 왜 이 결정이 필요했는지

## 선택지
1.
2.

## 결정
- 선택:
- 이유:

## 영향
-
```

#### comparison (비교)

```markdown
# [주제]

> 한 줄 요약

## 비교 대상

| 항목 | A | B |
|------|---|---|
|  |  |  |

## 결론
- 추천:
- 이유:
```

#### til (오늘 배운 것)

```markdown
# [주제]

> 한 줄 요약

## 질문
- 뭘 몰랐는지 / 뭘 궁금했는지

## 답변
- 핵심 설명 (간결하게)

## 예시
- 코드 또는 실제 사례

## 연관
- 관련 개념/키워드 링크
```

### 5. 저장 위치 선택

AskUserQuestion으로 저장 위치를 물어본다:

**질문 1: 저장 위치**
- "프로젝트 노트 (.ai/notes/)" — 이 프로젝트에서만 참고할 내용
- "개인 지식 (Obsidian)" — 프로젝트 무관한 범용 지식

**질문 2 (Obsidian 선택 시): 하위 폴더**
- "inbox" — 미분류, 나중에 정리
- "dev" — 개발 지식 (TIL, 가이드, 트러블슈팅)
- "projects" — 프로젝트별 노트

### 6. 파일 저장

선택에 따라 저장:

| 선택 | 저장 경로 |
|------|-----------|
| 프로젝트 노트 | `.ai/notes/YYYY-MM-DD-주제.md` |
| Obsidian | `~/obsidian-note/{폴더}/YYYY-MM-DD-주제.md` |

파일명 규칙:
- 형식: `YYYY-MM-DD-주제.md`
- 주제는 한글 2-4단어 케밥케이스 (예: `plugin-분리`, `캐시-전략`)
- 기존 파일 있으면 내용 추가 (덮어쓰기 X)

### 6. 태그 삽입

문서 상단 `> 한 줄 요약` 아래에 태그 추가:

```markdown
`#troubleshooting` `#react` `#캐시`
```

태그 규칙:
- 유형 태그 1개 (필수): `#troubleshooting`, `#guideline`, `#tech-memo`, `#decision`, `#comparison`, `#til`
- 주제 태그 1-3개: 기술/도구/영역 (예: `#react`, `#plugin`, `#git`)

### 8. 출력

```markdown
# 저장 완료

- 파일: [저장 경로/파일명.md]
- 위치: [프로젝트 | Obsidian/{폴더}]
- 유형: [자동 분류 결과]
- 태그: [태그 목록]
- 내용: [1줄 요약]
```
