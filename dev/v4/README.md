## AI 워크플로우

### 구조

```tsx
~/.claude/
├── [CLAUDE.md](http://claude.md/)                    # 글로벌 설정 (자동 적용)
├── agents/                      # (비어있음)
├── skills/
│   └── fe-principles/           # 구체적 패턴 (겹치는 부분 있지만, 고민중 (제거 여부))
└── commands/
	├── [architecture.md](http://architecture.md/)          # /architecture
	├── [recap.md](http://recap.md/)                 # /recap
	├── [review.md](http://review.md/)                # /review
	└── [organization.md](http://organization.md/)          # /organization
```

## CLAUDE.md 요약

```tsx
┌────────────────┬────────────────────────────────────┐
│ 섹션 │ 내용 │
├────────────────┼────────────────────────────────────┤
│ 언어 설정 │ 한국어 응답, 커밋 메시지 │
├────────────────┼────────────────────────────────────┤
│ Response Style │ 사설 없이 핵심만 │
├────────────────┼────────────────────────────────────┤
│ 작업 시작 전 │ 요구사항 불명확하면 객관식 질문 │
├────────────────┼────────────────────────────────────┤
│ 개념 질문 │ 짧게, 깊은 학습은 새 대화 │
├────────────────┼────────────────────────────────────┤
│ 워크플로우 │ Clarify → Plan → Implement │
├────────────────┼────────────────────────────────────┤
│ 코드 원칙 │ 11개 ID ([GOAL][SSOT]...) │
├────────────────┼────────────────────────────────────┤
│ 금지 패턴 │ 이른 추상화/파일 추출/상수 추출 등 │
├────────────────┼────────────────────────────────────┤
│ 네이밍 컨벤션 │ kebab-case, PascalCase 등 │
├────────────────┼────────────────────────────────────┤
│ 요청별 응답 │ 디버깅/비교/회고/설명 형식 │
└────────────────┴────────────────────────────────────┘
```
    

## Commands

```tsx
┌───────────────┬──────────────────────────────┬────────────────┐
│ 커맨드          │ 용도                          │ 시점            │
├───────────────┼──────────────────────────────┼────────────────┤
│ /architecture │ 기능 설계 (질문→옵션→산출물)       │ 복잡한 설계 시     │
├───────────────┼──────────────────────────────┼────────────────┤
│ /recap        │ 작업 정리/문서화 (md 저장)        │ 기능 완료 후      │
├───────────────┼──────────────────────────────┼────────────────┤
│ /review       │ 코드 리뷰 (Must/Should/Nit)     │ 코드 완료 후     │
├───────────────┼──────────────────────────────┼────────────────┤
│ /organization │ 작업 상태 저장                   │ 세션 종료 전     │  
└───────────────┴──────────────────────────────┴────────────────┘
```

### 설계 원칙

- CLAUDE.md = Skill (사고 기준 고정, 자동 적용)
- commands = 필요할 때 호출
- Hook = 없음 (필요 시 추가)
- Agent = 없음 (commands로 역할 전환)

---

### 워크플로우

Clarify → Plan → Implement

1. 요구사항 불명확 → 질문으로 명확화
2. 설계 필요 → /architecture
3. 구현 → 기본 대화 (CLAUDE.md 적용)
4. 리뷰 → /review
5. 정리 → /recap
6. 세션 종료 → /organization
