## AI 워크플로우

### 구조

```tsx
~/.claude/
├── [CLAUDE.md](http://claude.md/)                    # 글로벌 설정 (자동 적용)
├── agents/                      # (비어있음)
├── skills/
│   └── fe-principles/           # 구체적 패턴 (겹치는 부분 있지만, 고민중 (제거 여부))
└── commands/
	├── [architecture.md](http://architecture.md/)          # /architecture
	├── [recap.md](http://recap.md/)                 # /recap
	├── [review.md](http://review.md/)                # /review
	└── [organization.md](http://organization.md/)          # /organization
```    

### Commands

```tsx
┌───────────────┬──────────────────────────────┬────────────────┐
│ 커맨드          │ 용도                          │ 시점            │
├───────────────┼──────────────────────────────┼────────────────┤
│ /architecture │ 기능 설계 (질문→옵션→산출물)       │ 복잡한 설계 시     │
├───────────────┼──────────────────────────────┼────────────────┤
│ /recap        │ 작업 정리/문서화 (md 저장)        │ 기능 완료 후      │
├───────────────┼──────────────────────────────┼────────────────┤
│ /review       │ 코드 리뷰 (Must/Should/Nit)     │ 코드 완료 후     │
├───────────────┼──────────────────────────────┼────────────────┤
│ /organization │ 작업 상태 저장                   │ 세션 종료 전     │  
└───────────────┴──────────────────────────────┴────────────────┘
```

### 워크플로우

Clarify → Plan → Implement

1. 요구사항 불명확 → 질문으로 명확화 (Clarify)
2. 설계 필요 → /architecture
3. 구현 → 기본 대화 (CLAUDE.md 적용)
4. 리뷰 → /review
5. 정리 → /recap
6. 세션 종료 → /organization


# 추가 설명
- `.claude` 는 글로벌로 설정해서 범용적으로 사용할 용도 
  - 프로젝트별로 각각의 .claude 에 커스텀해서 세부 설정 적용 
- `.ai` 폴더는 claude 표준이 아닌 각 프로젝트 별로 ai 컨텍스트 등 을 잘 관리하기 위한 폴더


## `.ai` 폴더 개념 
### 구조
```sh
프로젝트/
├── .claude/
│   └── CLAUDE.md           # 프로젝트별 Claude 설정
│
└── .ai/
		├── context/            # 프로젝트 배경
		├── patterns/           # 구현 패턴
		├── prompts/            # 프롬프트 템플릿
		├── sessions/           # 작업 기록
		└── specs/              # [옵션] 스펙 문서 (스펙주도개발 시)
			├── features/
			└── templates/
```
###   폴더별 상세
```
  ┌───────────┬─────────────────┬────────────────────────────┬────────────────────────────────────────────────┐
  │   폴더    │      역할       │         언제 사용          │                   예시 파일                    │
  ├───────────┼─────────────────┼────────────────────────────┼────────────────────────────────────────────────┤
  │ context/  │ 프로젝트 배경   │ AI가 프로젝트 파악할 때    │ architecture.md, tech-stack.md, conventions.md │
  ├───────────┼─────────────────┼────────────────────────────┼────────────────────────────────────────────────┤
  │ patterns/ │ 구현 패턴       │ 구현 중 참조               │ filter.md, pagination.md, form-validation.md   │
  ├───────────┼─────────────────┼────────────────────────────┼────────────────────────────────────────────────┤
  │ prompts/  │ 프롬프트 템플릿 │ 반복 요청 시               │ feature-request.md, bug-fix.md                 │
  ├───────────┼─────────────────┼────────────────────────────┼────────────────────────────────────────────────┤
  │ sessions/ │ 작업 기록       │ /recap, /organization 결과 │ 2025-01-10-auth.md                             │
  ├───────────┼─────────────────┼────────────────────────────┼────────────────────────────────────────────────┤
  │ specs/    │ 기능 명세서     │ 스펙주도개발 시 (옵션)     │ filter-spec.md, pagination-spec.md             │
  └───────────┴─────────────────┴────────────────────────────┴────────────────────────────────────────────────┘
```

### 워크플로우 연동
```
  Clarify → Plan → Implement → Review → Document

      │         │         │         │         │
      ↓         ↓         ↓         ↓         ↓
   specs/   /architecture patterns/ /review  /recap
                                              ↓
                                         sessions/
```

### 사용 시나리오
```
┌─────────────────────────────┬───────────┬────────────────────────┐
│            상황             │ 참조 폴더 │          액션          │
├─────────────────────────────┼───────────┼────────────────────────┤
│ "필터 기능 만들어줘"        │ patterns/ │ 패턴 참조 → 구현       │
├─────────────────────────────┼───────────┼────────────────────────┤
│ "이 프로젝트 구조 알려줘"   │ context/  │ 아키텍처/기술스택 읽기 │
├─────────────────────────────┼───────────┼────────────────────────┤
│ "지난 작업 이어서"          │ sessions/ │ 마지막 세션 기록 읽기  │
├─────────────────────────────┼───────────┼────────────────────────┤
│ "버그 수정해줘"             │ prompts/  │ 버그 수정 템플릿 사용  │
├─────────────────────────────┼───────────┼────────────────────────┤
│ "필터 스펙 확인해줘" (옵션) │ specs/    │ 스펙 문서 읽기         │
└─────────────────────────────┴───────────┴────────────────────────┘
```

## 프로젝트 CLAUDE.md 추가하기
```md
## AI 컨텍스트 (.ai/)
.ai/
├── context/    # 프로젝트 배경
├── patterns/   # 구현 패턴
├── prompts/    # 프롬프트 템플릿
├── sessions/   # 작업 기록
└── specs/      # [옵션] 스펙 문서

### 사용 시점
- 프로젝트 파악 → `.ai/context/` 참조
- 구현 중 → `.ai/patterns/` 참조
- 반복 요청 → `.ai/prompts/` 참조
- `/recap`, `/organization` → `.ai/sessions/`에 저장

### 스펙주도개발 시 (옵션)
- 구현 전 → `.ai/specs/` 확인
- 스펙 없으면 작성 요청
```  

### 핵심 원칙
1. 필요할 때만 읽기 - 컨텍스트 효율 유지
2. 패턴 재사용 - 반복 구현은 patterns/에 정리
3. 기록 남기기 - 중요 작업은 sessions/에 저장
4. 프로젝트별 관리 - .ai/는 프로젝트 루트에
