# Global CLAUDE.md

## 언어 설정

- 항상 한국어로 응답
- 커밋 메시지도 한국어로 작성

## Response Style

사설 없이 핵심만. 장황한 설명 금지.

- 짧고 직접적으로
- 불필요한 인사/마무리 멘트 제거
- 코드로 보여줄 수 있으면 코드로

## 작업 시작 전

요구사항 불명확하면 객관식으로 질문해서 명확화.

## 개념 질문

구현 중 개념 질문은 짧게. 깊은 학습 필요하면 새 대화 권장.

## 워크플로우
1. 설계 필요 → `/architecture`
2. 구현 → 기본 대화
3. 리뷰 → `/review`
4. 정리 → `/recap`
5. 세션 종료 → `/organization`

## 프론트엔드 코드 원칙

React/TypeScript 기반. 목표: **수정 표면적 = 1**

### 우선순위 (상위일수록 강함)

| ID | 원칙 | 핵심 |
|----|------|------|
| [GOAL] | 변경 용이성 | 한 종류 변경 = 한 파일에서 끝 |
| [SSOT] | 단일 진실 공급원 | 정의 1곳, 사용 여러 곳 |
| [SRP] | 단일 책임 | 변경 이유 하나 |
| [COUP] | 응집↑ 결합↓ | 함께 바뀌는 것끼리, Page First |
| [ABS] | 추상화 깊이 일치 | 같은 레벨만, 세부는 위임 |
| [DECL] | 선언적 | What 선언, How는 하위로 |
| [TS] | TypeScript 엄격 | any 금지, strict, 추론 중심 |
| [READ] | 가독성 | 의도 드러나는 이름, 조기 반환 |
| [COG7] | 인지 부하 | 함수≤30줄, 파라미터≤3, 분기≤3 |
| [API] | 인터페이스 안정성 | 작고 명확, 옵션 객체로 확장 |
| [DRY] | DRY/KISS/YAGNI | 이른 일반화 금지 |

### 금지 패턴

- **이른 추상화**: 패턴이 2-3번 반복되고 명확해졌을 때만
- **이른 파일 추출**: 재사용 전까지 같은 파일 유지
- **성급한 상수 추출**: 여러 곳 사용 전까지 현재 위치 유지
- **과도한 에러 핸들링**: 일어나지 않을 시나리오 대비 금지
- **기능 추가**: 요청받은 것만

## 네이밍 컨벤션

| 대상 | 규칙 | 예시 |
|------|------|------|
| 디렉토리 | kebab-case | `task-detail/` |
| 컴포넌트 | PascalCase | `TaskHeader.tsx` |
| 훅 | use + camelCase | `useTaskDetail.ts` |
| API 함수 | fetch/post/update/delete + 명사 | `fetchUser` |

## 요청별 응답 스타일

| 요청 | 응답 형식 |
|------|-----------|
| "디버깅해줘" | 원인 분석 → 해결책 → 코드 |
| "비교해줘" | 표 형식, 장단점, 추천 1개 |
| "회고해줘" | 잘한 점, 개선점, 다음 액션 |
| "설명해줘" | 한 줄 요약 → 핵심 3개 → 예시 |
