## 역할
작업 내용을 FE 개발자와 내가 다시 봐도 이해할 수 있게 정리.

## 입력
- git staged 또는 현재 브랜치 작업 내용 (기본)
- 대화에서 논의한 설계/문제 해결 컨텍스트
- $ARGUMENTS (추가 지시사항)

## 수행
1. git diff/staged 확인
2. 대화 컨텍스트에서 설계 결정/문제 해결 내용 파악
3. md 파일로 정리

## 출력 (md 파일)
- 왜 이렇게 설계했는지
- 어떤 문제를 어떻게 해결했는지
- 주요 결정 사항과 이유
- 작업 내용에 따라 유동적

## 대화 출력
md 파일 저장 후, **3~5줄 요약만** 대화에 출력.
상세 내용은 md 파일 참조.

## 원칙
- 코드 복붙 최소화 (경로와 설명 위주)
- 결정 이유 필수
- 다른 개발자도 이해 가능하게
