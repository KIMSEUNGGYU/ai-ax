# Mid (3-4년차) 평가 기준

**적용 기준**: Senior 레벨 기준으로 평가
**합격선**: 60%

## 평가 관점

Mid 개발자에게 기대하는 것:
- 7가지 철학 원칙 이해 및 적용
- 트레이드오프 인지
- 유지보수 고려한 코드

## Phase별 기대 수준

### Phase 0: 요구사항
- 모든 Critical/High 요구사항 충족
- Medium도 대부분 충족

### Phase 1: 구조
- [COUP] 원칙 준수
- Page First 패턴 적용
- 명확한 도메인 분리

### Phase 2: 변경 용이성
- [GOAL] 원칙 이해
- 대부분의 시나리오에서 2곳 이내 수정
- 하드코딩 최소화

### Phase 3: 추상화
- ACC 체크리스트 대부분 통과
- 의도적인 추상화 결정
- 복잡한 훅 지양

### Phase 4: 가독성
- [READ] 의도 드러나는 네이밍
- [DECL] 선언적 코드 스타일
- [COG7] 30줄, 3파라미터 준수

### Phase 5: 타입
- any 0회 필수
- 타입 단언 최소화
- [SSOT] 타입 단일 정의

## 합격/불합격 판단

### 필수 통과 조건
1. Phase 0 Critical 100% 충족
2. any 사용 0회
3. tsc --noEmit 통과
4. 순환 참조 없음

### 점수 기준
- 코드 품질 (Phase 1-5) 60점 이상

## 피드백 톤

- 철학 원칙 기반 구체적 피드백
- "이 부분은 [GOAL] 원칙에 따라 ~하면 좋겠습니다"
- 트레이드오프 설명 요청
