# Staff+ (7년+) 평가 기준

**적용 기준**: Principal 레벨 기준으로 평가
**합격선**: 80%

## 평가 관점

Staff+ 개발자에게 기대하는 것:
- 시스템적 사고
- 모든 영역 완벽
- 조직 전체 기술 방향 리딩

## Phase별 기대 수준

### Phase 0: 요구사항
- 100% 충족은 기본
- 요구사항의 의도까지 파악
- 더 나은 대안 제시 가능

### Phase 1: 구조
- 대규모 확장 고려한 구조
- 모듈 간 명확한 계약
- 팀 단위 분업 가능한 구조

### Phase 2: 변경 용이성
- 모든 변경이 한 곳에서 완료
- 영향 범위 완전 격리
- 레거시 없이 깔끔한 구조

### Phase 3: 추상화
- 비즈니스 도메인 완벽 반영
- 미래 확장 고려한 추상화
- 불필요한 복잡성 제로

### Phase 4: 가독성
- 코드 = 문서
- 주석 없이도 의도 파악
- 팀원 누구나 이해 가능

### Phase 5: 타입
- 타입만으로 도메인 이해 가능
- 컴파일 타임 에러 최대화
- 런타임 에러 최소화

## 합격/불합격 판단

### 필수 통과 조건
1. 모든 Phase에서 감점 최소화
2. any 0회
3. 타입 단언 0회
4. 안티패턴 0개
5. 모든 [COG7] 기준 준수

### 점수 기준
- 코드 품질 (Phase 1-5) 80점 이상

## 피드백 톤

- 최고 수준 기대
- "Staff+로서 이 수준은 기대 이하입니다"
- 아키텍처 수준 개선 제안
- 시스템적 관점에서의 피드백
