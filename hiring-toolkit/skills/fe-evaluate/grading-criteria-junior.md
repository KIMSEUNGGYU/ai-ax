# Junior (1-2년차) 평가 기준

**적용 기준**: Mid 레벨 기준으로 평가
**합격선**: 50%

## 평가 관점

Junior 개발자에게 기대하는 것:
- 기본적인 코드 구조화 능력
- TypeScript 기초 이해
- 동작하는 코드 작성

## Phase별 기대 수준

### Phase 0: 요구사항
- Critical 요구사항 100% 충족 필수
- High/Medium은 일부 미충족 허용

### Phase 1: 구조
- 기본적인 폴더 분리
- 순환 참조 없음

### Phase 2: 변경 용이성
- 명백한 산탄총 수술 패턴 없음
- 일부 하드코딩 허용

### Phase 3: 추상화
- 과도한 추상화보다 직관적인 코드 선호
- 복잡한 훅 패턴 불필요

### Phase 4: 가독성
- 기본적인 네이밍 규칙 준수
- 함수 길이 50줄 이내 허용

### Phase 5: 타입
- **any 사용 0회 필수**
- 기본적인 타입 정의
- tsc 통과

## 합격/불합격 판단

### 필수 통과 조건
1. Phase 0 Critical 100% 충족
2. any 사용 0회
3. tsc --noEmit 통과

### 점수 기준
- 코드 품질 (Phase 1-5) 50점 이상

## 피드백 톤

- 격려와 함께 개선점 제시
- "좋은 시작점이지만, ~하면 더 좋을 것"
- 구체적인 학습 자료 추천
