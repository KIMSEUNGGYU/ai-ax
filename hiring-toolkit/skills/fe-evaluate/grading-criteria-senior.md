# Senior (5-6년차) 평가 기준

**적용 기준**: Staff+ 레벨 기준으로 평가
**합격선**: 70%

## 평가 관점

Senior 개발자에게 기대하는 것:
- 7가지 철학 원칙 완벽 준수
- "왜?" 설명 가능
- 팀 전체 코드 품질 리딩

## Phase별 기대 수준

### Phase 0: 요구사항
- 모든 요구사항 충족
- 명세에 없는 엣지 케이스까지 고려

### Phase 1: 구조
- [COUP] 완벽한 응집도
- 확장 가능한 폴더 구조
- 의존성 방향 명확

### Phase 2: 변경 용이성
- [GOAL] 모든 변경 시나리오에서 1곳 수정
- 명확한 경계와 인터페이스
- 확장에 열림, 수정에 닫힘

### Phase 3: 추상화
- ACC 전체 통과
- 적절한 추상화 수준
- 비즈니스 로직과 UI 완전 분리

### Phase 4: 가독성
- 완벽한 네이밍
- 코드가 문서 역할
- [COG7] 엄격 준수

### Phase 5: 타입
- any 0회 필수
- 타입 단언 0회 권장
- 완벽한 타입 안전성

## 합격/불합격 판단

### 필수 통과 조건
1. Phase 0 모든 요구사항 충족
2. any 사용 0회
3. 타입 단언 2회 이내
4. 순환 참조 없음
5. 모든 함수 30줄 이내

### 점수 기준
- 코드 품질 (Phase 1-5) 70점 이상

## 피드백 톤

- 높은 기준 적용
- "Senior로서 ~한 부분이 아쉽습니다"
- 구조적 개선 제안
- "왜 이렇게 구현했는지" 설명 요청
