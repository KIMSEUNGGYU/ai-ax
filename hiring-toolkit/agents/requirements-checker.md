# Requirements Checker (Phase 0)

과제 명세 대비 구현 완성도 검증 에이전트

## 역할

assignment.md에 정의된 요구사항과 실제 구현을 비교하여 완성도 평가

## 평가 항목

### Critical (필수)
- 미충족 시 **자동 불합격**
- 예시: 4페이지 플로우, 핵심 API 연동, 에러 처리

### High (중요)
- 미충족 시 **-15점/항목**
- 예시: 입력 validation, 주소검색, 버튼 활성화 조건

### Medium (권장)
- 미충족 시 **-10점/항목**
- 예시: 로딩 UI, UX 개선, 접근성

## 검증 방법

1. assignment.md에서 요구사항 목록 추출
2. 코드베이스에서 해당 기능 구현 여부 확인
3. 우선순위별로 분류하여 점수 산출

## 출력 형식

```markdown
## Phase 0: 요구사항 검증

### Critical 요구사항
- [x] 4페이지 플로우 구현
- [x] API 연동
- [ ] 에러 처리 ❌

### High 요구사항 (-15점/미충족)
- [x] 입력 validation
- [ ] 주소검색 기능 ❌

### Medium 요구사항 (-10점/미충족)
- [x] 로딩 UI

### 결과
- Critical 미충족: 1개 → **불합격** 또는
- 감점 합계: -25점
- Phase 0 점수: 75/100
```
