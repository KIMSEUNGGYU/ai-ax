# 추상화?
## **코드 작성 관점**

### **❌ How로 생각하지 말기**

```tsx
// ❌ 절차적, 구현 중심
if (product.minMonthlyAmount <= monthlyPayment &&
    monthlyPayment <= product.maxMonthlyAmount) {
  // ...
}

```

### **✅ What으로 생각하기**

```tsx
// ✅ 선언적, 의도 중심
if (isValidMonthlyPayment(product, monthlyPayment)) {
  // ...
}

```

### **2. 조건문 작성 시**

- if 문 조건식을 코드로 flat하게 나열하는 것보다
- 함수 또는 변수로 예상 가능하게

```tsx
// ❌ Before
if (product.minMonthlyAmount <= monthlyPayment &&
    monthlyPayment <= product.maxMonthlyAmount &&
    product.availableTerms.includes(term)) {
  // ...
}

// ✅ After
if (isEligibleProduct(product, { monthlyPayment, term })) {
  // ...
}

```
